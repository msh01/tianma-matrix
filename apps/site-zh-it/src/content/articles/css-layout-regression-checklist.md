CSS 布局问题很少在“正常文案、正常屏幕、正常数据”里暴露。真正让页面难看的，是长标题、窄屏、翻译后的按钮、滚动条、弹窗和 hover 状态。

我做布局回归时会先看几个固定尺寸：手机宽度、平板宽度、普通笔记本、宽屏桌面。只看 1440 宽度没有意义，因为很多问题恰好被宽屏掩盖。

## 长文本是最好的测试数据

标题、按钮、分类名、作者名都应该塞一次长文本。能换行的地方要自然换行，不能换行的地方要有截断策略。最怕的是文字把卡片撑开，导致整行网格高度混乱。

论坛式列表尤其要注意。话题标题可以截断，但摘要不能覆盖后面的回复数和浏览量。右侧数字列宽度应该稳定，不要因为某个浏览量变成五位数就乱掉。

## 固定栏要测滚动

顶部固定导航看起来简单，实际常常挡住锚点、遮住内容或影响移动端输入框。改 header 高度以后，主内容的 `padding-top` 也要跟着确认。

如果页面有侧边栏，还要看滚动区域是否正确。侧边栏滚动和主内容滚动混在一起，会让用户很难找回位置。

## 状态也属于布局

hover、focus、active、loading 都可能改变尺寸。按钮边框变粗、字体加粗、图标出现，都可能让布局抖动。

一个简单原则是：状态变化只改变颜色、阴影或透明度，尽量不要改变元素占用空间。布局稳定，页面才像一个认真维护的产品。
