页面出问题时，很多人第一反应是打开 DevTools 看 Elements。但如果页面是服务端渲染或静态生成，第一步更应该看原始 HTML。

Elements 面板展示的是浏览器解析和客户端脚本运行后的结果。View Source 或直接请求 HTML，才能看到服务端到底输出了什么。

## 先确认内容是否在 HTML 里

如果文章标题、meta description、列表项已经在 HTML 中，说明生成阶段基本正常。问题可能在 CSS、客户端脚本或浏览器解析。

如果 HTML 里没有内容，那就要回到数据获取、构建路径、模板条件判断。不要在客户端状态里找不存在的问题。

## 再看资源是否加载

HTML 正常但页面样式错乱，通常看 CSS 和 JS 资源。路径是否 404，MIME 类型是否正确，base 路径是否被错误拼接，这些都很常见。

静态站部署到子路径时尤其要注意。开发环境根路径正常，不代表生产路径也正常。

## 最后看客户端增强

搜索、菜单、评论这些交互可能依赖客户端脚本。它们失败时，不应该影响正文 HTML。排查时要把核心内容和增强功能分开。

调试渲染问题时，分层很重要：服务端输出、资源加载、客户端运行。层次清楚，问题就不会混成一团。
